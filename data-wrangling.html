In this notebook, we focus on children who have died during migration. To do so, we rely on data collected by the organization United Against Racism. This work follows on from the study day “Geovisualizations of Migration: From Tables to Maps”, jointly organized by UMR 7301 Migrations internationales, espaces et sociétés (MIGRINTER) and UMR 5194 Laboratoire de science sociale (PACTE), with the support of the research initiative (Carto)graphies and (Géo)visualisation of data – AR09 (AR9) of the CNRS GdR MAGIS.

The UNITED List of Refugee Deaths

The UNITED List of Refugee Deaths summarises all death cases since 1993. The data are collected through independent research, as well as information received from a network of 550 organisations in 48 countries, and from local experts, journalists, and researchers in the field of migration. An updated list is published annually in the first week of June.

library(readxl)
data <- data <- read_excel("unitedMorts.xlsx", col_types = "text")

In the dataset, there is a column “name, gender, age” that contains incomplete information about the presence of children in events that caused the deaths of migrants. This notebook aims to extract this information in order to determine the number of children who died during migration.

Data wrangling

We create a field that will be used to identify events involving children.

data$`Number of deaths` <- as.numeric(data$`Number of deaths`)
data$children_flag <- FALSE
data$children <- NA

Step 1: Identify events involving children, babies, and pregnant women.

  • We keep the rows that contain the following character strings.
str <- c("children", "minor", "child", "boy", "girl", "teen", "baby",
"infant", "babies", " son", "toddler")
pattern1 =  paste(tolower(str), collapse = "|")
cond1 <- grepl(pattern1, tolower(data$`name, gender, age`))
  • We are trying to retrieve ages ≤ 17.
# Pattern where age is at the end of the string.
pattern2 <- ",\\s*(?:[0-9]|1[0-7])\\)$"
cond2 <- grepl(pattern2, tolower(data$`name, gender, age`))
# Pattern for age ranges like "xx-yy" with xx <= 17
pattern3 <- "\\b(?:[0-9]|1[0-7])-\\d+\\b"
cond3 <- grepl(pattern3, data$`name, gender, age`)
  • We apply the conditions.
data$children_flag <- cond1 | cond2 |cond3

Step 2: Estimate the number of deaths.

  • If children_flag is FALSE, in that case there are zero children.
data$children[!data$children_flag] <- 0
  • If “children_flag” is TRUE and “Number of deaths” equals 1, then there is one child.
data$children[data$children_flag & data$`Number of deaths` == 1] <- 1
  • We retrieve the value between “incl” and “child”, “boy” or “girl” or “infant”…
na_idx <- is.na(data$children)
data$children[na_idx] <- as.numeric(
  sub(".*(?:incl|inck|inck.|\\(|;)\\s*(\\d+)\\s*(?:child|boy|girl|minor|young child|baby|babies|young girls|infant|teenage).*", "\\1",
      tolower(data$`name, gender, age`[na_idx]))
)

  idx <- na_idx & grepl(pattern, tolower(data$`name, gender, age`))
  value <- lookup[[pattern]]
  if (is.function(value)) {
    data$children[idx] <- value(data$`Number of deaths`[idx])

    data$children[idx] <- value
  }

For entries containing plural nouns without any further distinction (e.g. children, infants…), set the value to 2.
